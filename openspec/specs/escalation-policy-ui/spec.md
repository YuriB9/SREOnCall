# Escalation Policy UI

## Purpose

Управление политиками эскалации: список с бейджем «По умолчанию», визуальный конструктор шагов (вертикальный степпер с выбором расписания и таймаутом), назначение политики по умолчанию для тенанта.

## Requirements

### Requirement: Список политик эскалации

Система ДОЛЖНА (SHALL) отображать список всех политик эскалации текущего тенанта, получаемых через `GET /api/escalations/v1/{tenant}/policies`. Для каждой политики ДОЛЖНЫ отображаться: название, количество шагов и признак того, является ли она политикой по умолчанию для тенанта. Система ДОЛЖНА позволять создавать новую политику через кнопку «Новая политика» и удалять не-дефолтную политику через действие удаления.

#### Scenario: Список политик отображается

- **WHEN** пользователь переходит на `/[tenant]/escalations`
- **THEN** система ДОЛЖНА отобразить все политики эскалации с названием, количеством шагов и бейджем «По умолчанию» на активной политике по умолчанию

#### Scenario: Удаление не-дефолтной политики

- **WHEN** пользователь нажимает «Удалить» на не-дефолтной политике
- **THEN** система ДОЛЖНА показать диалог подтверждения и при подтверждении удалить политику через `DELETE /api/escalations/v1/{tenant}/policies/{id}`

---

### Requirement: Визуальный конструктор шагов политики эскалации

Система ДОЛЖНА (SHALL) рендерить редактор политики эскалации в виде вертикального степпера. Каждый шаг ДОЛЖЕН содержать: номер шага, выпадающий список расписаний (заполненный из `GET /api/schedules/v1/{tenant}/schedules`) и поле ввода таймаута; значение таймаута передаётся в API в поле `timeout_seconds` (положительное целое). Пользователь ДОЛЖЕН иметь возможность добавлять, удалять и переупорядочивать шаги с помощью кнопок со стрелками вверх/вниз.

Поскольку бэкенд не предоставляет `PUT /policies/{id}`, сохранение изменений существующей политики ДОЛЖНО выполняться заменой: `POST /api/escalations/v1/{tenant}/policies` с новой версией → если старая политика была политикой по умолчанию, `PUT /default-policy` с новым `policy_id` → `DELETE /policies/{oldId}`. При сбое POST ничего не теряется; при сбое промежуточных шагов пользователю показывается ошибка (известное ограничение: операция не атомарна).

#### Scenario: Добавление шага

- **WHEN** пользователь нажимает «Добавить шаг» в редакторе политики
- **THEN** система ДОЛЖНА добавить новую строку шага с пустым выпадающим списком расписаний и таймаутом по умолчанию

#### Scenario: Удаление шага

- **WHEN** пользователь нажимает кнопку удаления на шаге (минимум 1 шаг должен оставаться)
- **THEN** система ДОЛЖНА удалить этот шаг и перенумеровать последующие шаги

#### Scenario: Переупорядочивание шагов

- **WHEN** пользователь нажимает стрелку «Вверх» на шаге 2
- **THEN** шаги 1 и 2 ДОЛЖНЫ поменяться местами

#### Scenario: Сохранение изменений существующей политики

- **WHEN** пользователь сохраняет отредактированную политику
- **THEN** система ДОЛЖНА создать новую политику через POST, перенести признак «по умолчанию» (если был) через `PUT /default-policy` и удалить старую политику; список обновляется и показывается toast об успехе

#### Scenario: Сохранение с незаполненным шагом

- **WHEN** выпадающий список расписания одного из шагов пуст и пользователь нажимает «Сохранить»
- **THEN** система ДОЛЖНА отобразить встроенную ошибку валидации на незаполненном шаге и НЕ ДОЛЖНА отправлять запрос

---

### Requirement: Назначение политики по умолчанию

Система ДОЛЖНА (SHALL) позволять администратору назначить любую политику политикой тенанта по умолчанию, нажав «Сделать по умолчанию» на политике. Это ДОЛЖНО вызвать `PUT /api/escalations/v1/{tenant}/default-policy` с телом `{ "policy_id": "<id>" }`. Текущая политика по умолчанию определяется через `GET /api/escalations/v1/{tenant}/default-policy`; бейдж «По умолчанию» отображается на соответствующей политике списка.

#### Scenario: Установка новой политики по умолчанию

- **WHEN** администратор нажимает «Сделать по умолчанию» на не-дефолтной политике
- **THEN** система ДОЛЖНА выполнить `PUT /default-policy` с `policy_id` выбранной политики, обновить список и показать бейдж «По умолчанию» на новой политике, убрав его с предыдущей
