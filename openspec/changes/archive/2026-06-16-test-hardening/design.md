## Context

Аудит области 8 (`docs/audit/08-testing.md`) выделил три остаточных пробела после CH01 (T1/T2): нет детекции утечек горутин (T3), не покрыты дешёвые и важные юниты (T4), нет `t.Parallel()`/table-driven и есть латентный `go vet`-nil-deref в хендлер-тестах (T5). Чейндж — чисто тестовый: трогает только `*_test.go` и dev-зависимости, продакшн-код, API, схему БД и события RabbitMQ не меняет.

## Goals / Non-Goals

**Goals:**
- Ловить утечки горутин в пакетах с фоновыми задачами (`pkg/amqp`, консьюмеры, `monitor`, sweeper `tenantcache`).
- Закрыть приоритетные юнит-пробелы T4, включая регресс-гард D1 на store эскалации настоящим integration-тестом.
- Перевести матричные тесты на table-driven с именованными подтестами и добавить `t.Parallel()`; убрать `go vet`-замечания httpresponse.

**Non-Goals:**
- Не переписывать существующие корректные тесты ради покрытия ради покрытия (аудит явно: «не всё подряд»).
- Не менять продакшн-код. Если тест вскрывает реальный баг вне объёма — записать в backlog соответствующего чейнджа, не править здесь.
- Не вводить capability-дельту (нет наблюдаемого изменения поведения продукта).

## Decisions

- **goleak через `TestMain`, а не `defer goleak.VerifyNone(t)`.** `VerifyTestMain(m)` покрывает все тесты пакета одной точкой и не требует правки каждого теста; библиотечные фоновые горутины гасятся `goleak.IgnoreCurrent()` при необходимости. Альтернатива (per-test `defer`) отвергнута — шумнее и легко забыть в новом тесте.
- **Store эскалации — integration-тест под тегом `integration`** (паттерн `scheduling/store_integration_test.go`): живой Postgres из docker-compose, `t.Skip` без `DB_DSN`. CAS-конфликт D1 проверяется параллельным запуском `AdvanceEscalationState` на одной строке: ровно один успех, остальные → `errs.ErrConflict`. Альтернатива (мок pgx) отвергнута — D1 — это гонка на уровне SQL `WHERE current_tier=$exp`, мок её не воспроизведёт.
- **`monitor.step()` — юнит на интерфейсном моке** Store/escalator (без БД): тикерный `Run` покрывает goleak, а бизнес-логика отбора истёкших — быстрый юнит.
- **dispatcher (C5)** — проверяем, что ретрай-ожидание прерывается по `ctx` (отменённый контекст → быстрый возврат, без `time.Sleep` на всю задержку).
- **Нет capability-дельты, архив `--skip-specs`** — прецедент `harden-auth-shell` (no-delta infra/tooling).

## Risks / Trade-offs

- [goleak ложно срабатывает на фоновых горутинах зависимостей] → `goleak.IgnoreCurrent()` в `TestMain` и точечные `IgnoreTopFunction` для известных пулов (pgx/amqp091).
- [integration-тест требует Postgres, в CI без сервиса скипается] → `t.Skip` без `DB_DSN`; прогон локально из docker-compose перед мержем + в integration-джобе CI.
- [paralleltest-линтер (baseline CH01) подсветит новые `t.Parallel()` с захватом переменной цикла] → Go 1.26: переменная цикла уже per-iteration; всё равно проверить отсутствием шаринга в подтестах.

## Migration Plan

Откат тривиален — изменения только в тестах и `go.mod`/`go.sum` (dev-dep). Деплой продакшна не затрагивается. Совместимость событий/схемы БД не нарушается.
