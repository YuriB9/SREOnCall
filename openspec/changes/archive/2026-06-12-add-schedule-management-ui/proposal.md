## Why

Бэкенд `services/scheduling` уже предоставляет полный CRUD расписаний:
`POST /api/schedules/v1/{tenant}/schedules` (создание), `PATCH .../schedules/{id}`
(редактирование), `DELETE .../schedules/{id}` (удаление) — см.
`internal/handler/handler.go` (`CreateSchedule`, `PatchSchedule`, `DeleteSchedule`).

Однако во фронтенде создать расписание невозможно: на странице расписаний
([SchedulesPage.tsx](../../../frontend/src/pages/SchedulesPage.tsx)) нет ни кнопки
«Создать расписание», ни формы. Слой API
([schedules.ts](../../../frontend/src/api/schedules.ts)) умеет только читать список
(`useSchedules`) и работать с переопределениями. Пустое состояние буквально
отсылает пользователя к API: «Расписаний нет. Создайте первое через API». То есть
новый тенант не может начать пользоваться платформой без ручных вызовов API/seed-скрипта.

Спецификация `oncall-schedule-ui` описывает только просмотр расписаний и создание
переопределений, но не их создание/редактирование/удаление.

## What Changes

- В слой API ([schedules.ts](../../../frontend/src/api/schedules.ts)) добавляются
  мутации `useCreateSchedule`, `useUpdateSchedule`, `useDeleteSchedule`, бьющие в уже
  существующие эндпоинты бэкенда; при успехе инвалидируется `scheduleKeys(tenant).list()`.
- Добавляется модальное окно `ScheduleFormModal` для создания и редактирования
  расписания с полями: название, часовой пояс, продолжительность смены
  (`shift_duration`, ISO-8601, с пресетами P1D/P7D/P14D/PT8H/PT12H), дата старта ротации
  (`start_date`, YYYY-MM-DD) и упорядоченная ротация инженеров (выбор из участников
  тенанта `useMembers`, с возможностью добавлять/удалять/менять порядок).
- На странице расписаний добавляются точки входа: кнопка «Создать расписание»
  (всегда доступна, в т.ч. в пустом состоянии вместо текста «создайте через API»),
  а в строке/карточке каждого расписания — действия «Редактировать» и «Удалить»
  (удаление с подтверждением).
- Ошибки валидации бэкенда (`422` с `missing_fields` / `error`) показываются
  встроенно в модальном окне.

Изменения только во фронтенде; контракт API и бэкенд не меняются.

### Modified Capabilities
- `oncall-schedule-ui`: добавляется требование «Создание и управление расписаниями»
  (создание, редактирование, удаление через UI), а пустое состояние Gantt-диаграммы
  уточняется — вместо «создайте через API» предлагается кнопка создания.

## Impact

- **Фронтенд:**
  - `frontend/src/api/schedules.ts` — мутации create/update/delete расписания.
  - `frontend/src/pages/ScheduleFormModal.tsx` — новый компонент формы (новый файл).
  - `frontend/src/pages/SchedulesPage.tsx` — кнопка «Создать расписание», действия
    «Редактировать»/«Удалить» по расписанию, обновление пустого состояния.
  - При необходимости `frontend/src/api/types.ts` — input-типы для мутаций.
- **Бэкенд / API / БД / события:** не затрагиваются (эндпоинты уже существуют).
