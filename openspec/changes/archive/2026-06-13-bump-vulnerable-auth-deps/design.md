# Design — bump-vulnerable-auth-deps

## Контекст

Чейндж закрывает находку DC1 ([docs/audit/11-dependencies-ci.md:26](../../../docs/audit/11-dependencies-ci.md))
— две достижимые уязвимости в пути аутентификации. Это infra/tooling-изменение:
обновление версий зависимостей + ужесточение CI-гейта. Продуктовой логики и контрактов
шины оно не касается, поэтому спек-дельт нет (прецедент `harden-auth-shell`).

## Решение

1. **`jwt/v5` → `v5.2.2`** — прямой бамп в `pkg/go.mod` (где он `direct`). В четырёх
   auth-сервисах он `indirect`, подтянется через `go mod tidy` после обновления `pkg`
   в воркспейсе (`go.work` использует локальные модули, но `go.sum` каждого модуля
   обновляется независимо).
2. **`jwkset` → `v0.11.0` через `keyfunc/v3` → `v3.8.0`.** Указанная в аудите версия
   `jwkset v0.6.0` оказалась **ретрактнута автором** (гонка в refresh-горутине, PR #42)
   и исключена из листинга версий; более того, текущая `keyfunc/v3 v3.3.5` сама попадает
   в ретракт-диапазон `[v3.0.0, v3.3.5]` именно по GO-2025-3376. Поэтому корректный фикс —
   бамп `keyfunc/v3 → v3.8.0` (последняя стабильная, вне ретрактов), которая в своём
   `go.mod` требует неретракт-фикс `jwkset v0.11.0` и `jwt/v5 v5.2.2`. `jwkset`
   фиксируется в `pkg/go.mod` явно (`@v0.11.0`), чтобы MVS не выбрал ретракт-версию.
3. **`go mod tidy`** в каждом задетом модуле; проверка `git diff --exit-code` по
   `go.mod`/`go.sum` (тот же гейт честности, что в CI).
4. **`govulncheck ./...`** в `pkg` и каждом сервисе — обе достижимые уязвимости
   (GO-2025-3553, GO-2025-3376) должны исчезнуть. Проверено: `0 vulnerabilities`
   во всех 6 модулях.
5. **CI-гейт:** снять `continue-on-error: true` с джоба `govulncheck`
   ([.github/workflows/ci.yml:130](../../../.github/workflows/ci.yml)) и убрать
   поясняющий комментарий про «временно информационный» — теперь это блокирующий гейт.

## Совместимость при деплое

- Изменения версий — патч (`v5.2.1 → v5.2.2`) и минор (`v0.5.19 → v0.6.0`); публичный
  API библиотек стабилен, изменений в коде сервисов не требуется.
- Контракты RabbitMQ, схемы БД и HTTP API не меняются → rolling-деплой безопасен,
  старые сообщения в очередях и существующие токены не затрагиваются.
- Поведение `jwkset@v0.11.0`: устаревшие ключи теперь корректно вытесняются из кэша
  при refresh — это **усиление** гарантий ротации (отозванный ключ перестаёт быть
  валидным), деградации совместимости нет.

## Отклонённые альтернативы

- **Обновлять только `pkg`, не трогая `go.sum` сервисов.** Отклонено: `govulncheck`
  в CI гоняется per-module по `go.sum` каждого сервиса — без их `tidy` гейт остался
  бы красным.
- **Подчищать заодно все недостижимые уязвимости из отчёта govulncheck.** Отклонено:
  вне объёма CH02 (зафиксирован на DC1); общий апдейт дерева — отдельная задача.
- **Оставить `govulncheck` информационным.** Отклонено: прямое требование хэндоффа
  CH01 — после фикса DC1 сделать джоб блокирующим, иначе следующий регресс не словится.
