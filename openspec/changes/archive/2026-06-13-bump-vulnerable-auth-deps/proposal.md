## Why

`govulncheck ./...` подтвердил **две достижимые** уязвимости в пути аутентификации
(находка DC1, [docs/audit/11-dependencies-ci.md](../../../docs/audit/11-dependencies-ci.md)).
Это вызываемый, а не транзитивно-висящий код — валидация JWT стоит на входе каждого
защищённого запроса:

- **GO-2025-3553** — `github.com/golang-jwt/jwt/v5@v5.2.1`: чрезмерная аллокация при
  разборе специально сформированного токена → DoS неаутентифицированным запросом
  (достижимо из `auth.Middleware → jwt.ParseWithClaims`).
- **GO-2025-3376** — `github.com/MicahParks/jwkset@v0.5.19` (транзитивно через
  `keyfunc/v3`): refresh JWK Set только дополняет кэш и не удаляет устаревшие ключи →
  отозванный/ротированный в Keycloak ключ может остаться валидным (достижимо из
  `auth.Middleware → keyfunc.NewDefault`).

Обе чинятся бампом версии без мажорной миграции. Приоритет — немедленный.

## What Changes

- Бамп `github.com/golang-jwt/jwt/v5` → `v5.2.2` во всех модулях, где он в дереве
  (`pkg` напрямую; 4 auth-сервиса транзитивно).
- Фикс `jwkset`: версия `v0.6.0` из аудита **ретрактнута автором** (гонка в refresh,
  PR #42), поэтому фикс GO-2025-3376 подтягивается через бамп `keyfunc/v3 v3.3.5 → v3.8.0`
  (текущая `v3.3.5` сама в ретракт-диапазоне GO-2025-3376), который тянет неретракт-фикс
  `jwkset v0.5.19 → v0.11.0`.
- `go mod tidy` в каждом задетом модуле; синхронизация `go.sum`.
- Перепроверка `govulncheck ./...` — обе достижимые уязвимости должны исчезнуть.
- **Хэндофф CH01:** снять `continue-on-error: true` с джоба `govulncheck` в
  [.github/workflows/ci.yml](../../../.github/workflows/ci.yml) — сделать его блокирующим гейтом
  (теперь, когда DC1 закрыт, любой будущий регресс уязвимости валит CI).

Изменений API, событий RabbitMQ или схем БД нет — поведение сервисов не меняется.

## Impact

- **Затронутые сервисы:** `pkg` (прямой потребитель `jwt/v5`), а транзитивно —
  `services/scheduling`, `services/incident`, `services/escalation`,
  `services/notification` (импортируют `pkg/auth`). `services/ingestion` auth не
  использует и не затронут.
- **События RabbitMQ:** не затронуты (нет изменений контрактов шины).
- **Зависимости:** `go.mod`/`go.sum` перечисленных модулей; `jwt/v5 v5.2.1 → v5.2.2`,
  `keyfunc/v3 v3.3.5 → v3.8.0`, `jwkset v0.5.19 → v0.11.0`.
- **CI:** джоб `govulncheck` в `.github/workflows/ci.yml` становится блокирующим.
- **Совместимость:** изменения патч/минор, API библиотек стабилен — пересборки кода
  не требуется. Никаких ADR (нет значимого арх-решения) и спек-дельт.
