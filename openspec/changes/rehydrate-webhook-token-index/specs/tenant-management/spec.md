## ADDED Requirements

### Requirement: Регидрация Redis-индекса вебхук-токенов при старте

Сервис scheduling ДОЛЖЕН (SHALL) при старте восстанавливать Redis-индекс `oncall:tokens:{hash}` из таблицы `scheduling.tenant_webhook_tokens`, чтобы ранее выданные действующие токены продолжали резолвиться сервисом ingestion после перезапуска или сброса Redis (Redis работает без персистентности).

Регидрация выполняется один раз при инициализации, после применения миграций и установления подключения к Redis, и ДОЛЖНА быть идемпотентной: для каждого хранящегося хэша устанавливается `HSET oncall:tokens:{hash} tenant_id {slug}`; повторный запуск приводит к тому же состоянию.

Если Redis недоступен на старте, сервис ДОЛЖЕН запускаться в прежнем режиме (индекс токенов отключён, в журнал пишется предупреждение), а регидрация ДОЛЖНА пропускаться без прерывания запуска. Сбой загрузки отдельных записей не ДОЛЖЕН прерывать запуск сервиса.

Регидрация НЕ удаляет из Redis записи, отсутствующие в Postgres (очистка осиротевших ключей выходит за рамки требования); отзыв токена по-прежнему удаляет запись через существующий механизм (`HDEL`).

#### Scenario: Восстановление токенов после сброса Redis

- **WHEN** в `scheduling.tenant_webhook_tokens` есть действующий токен, а соответствующий ключ `oncall:tokens:{hash}` в Redis отсутствует (например, после перезапуска Redis), и сервис scheduling стартует
- **THEN** ключ `oncall:tokens:{hash}` восстанавливается со значением `tenant_id`, равным slug тенанта, и последующий вебхук ingestion с этим токеном резолвится в тенанта (не возвращает HTTP 401)

#### Scenario: Идемпотентность при повторном старте

- **WHEN** сервис scheduling стартует повторно, а индекс токенов в Redis уже наполнен
- **THEN** записи перезаписываются тем же значением, итоговое состояние индекса не меняется, дублирующихся или конфликтующих записей не возникает

#### Scenario: Redis недоступен на старте

- **WHEN** при старте scheduling не удаётся подключиться к Redis
- **THEN** сервис запускается с отключённым индексом токенов и предупреждением в журнале, регидрация пропускается, запуск не прерывается
