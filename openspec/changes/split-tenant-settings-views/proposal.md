## Why

Сейчас все настройки тенанта (webhook-токены, конфигурация уведомлений, участники команды) свалены в одну длинную прокручиваемую страницу `/[tenant]/settings`. По мере роста числа разделов это становится неудобным: нет прямых ссылок на конкретный раздел, нельзя быстро переключаться между ними, всё грузится сразу. Разделение на отдельные экраны с подменю в навигации делает структуру понятнее и масштабируемее.

## What Changes

- Пункт «Настройки» в боковой навигации становится разворачиваемым: при клике под ним раскрывается подменю с тремя кнопками-ссылками — «Webhook-токены», «Конфигурация уведомлений», «Участники команды».
- Каждая кнопка ведёт на собственный маршрут:
  - `/[tenant]/settings/webhook-tokens`
  - `/[tenant]/settings/notifications`
  - `/[tenant]/settings/members`
- Единая страница `TenantSettingsPage`, рендерящая все три секции стопкой, разбивается на три отдельных экрана; каждый показывает только свою секцию.
- Маршрут `/[tenant]/settings` (без подраздела) делает редирект на первый подраздел (`webhook-tokens`), сохраняя обратную совместимость старой ссылки.
- Ограничение доступа только для администраторов и скрытие пункта «Настройки» для не-админов сохраняются и распространяются на все три подмаршрута.

## Capabilities

### New Capabilities

(нет)

### Modified Capabilities

- `tenant-settings-ui`: разделы настроек становятся отдельными экранами с собственными маршрутами вместо одной страницы; добавляется требование к раскрывающемуся подменю навигации и редиректу с базового маршрута настроек.
- `frontend-shell`: пункт «Настройки» в боковой навигации становится разворачиваемой группой с тремя подпунктами.

## Impact

- Фронтенд: `frontend/src/layouts/GlobalLayout.tsx` (раскрывающийся пункт навигации), `frontend/src/routes.tsx` (новые вложенные маршруты + редирект), `frontend/src/pages/TenantSettingsPage.tsx` (разбивается на отдельные страницы-экраны для каждой секции).
- Без изменений в бэкенде и API: используются те же эндпоинты (`webhook-tokens`, `notification-config`, `members`).
- Без изменений событий RabbitMQ и схем БД.
- Возможное обновление E2E-тестов навигации в `tests/e2e`, если они полагаются на единую страницу настроек.
