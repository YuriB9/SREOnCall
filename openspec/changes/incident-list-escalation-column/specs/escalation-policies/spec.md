## ADDED Requirements

### Requirement: Массовое получение состояний эскалации

Сервис escalation ДОЛЖЕН (SHALL) предоставлять GET `/api/escalations/v1/{tenant}/incidents/state?incident_ids=<id1,id2,...>`, возвращающий массив состояний эскалации для перечисленных инцидентов текущего тенанта. Каждый элемент ДОЛЖЕН содержать как минимум `incident_id`, `current_tier` и `status` (`active | acknowledged | resolved | exhausted`).

Инциденты из запроса, для которых состояние эскалации отсутствует, в ответ НЕ включаются (отсутствие элемента трактуется потребителем как «эскалация не запускалась»); это не является ошибкой. Эндпоинт ДОЛЖЕН возвращать только состояния, принадлежащие тенанту из URL; идентификаторы чужих тенантов не раскрываются.

Пустой или отсутствующий `incident_ids` ДОЛЖЕН приводить к пустому массиву (или HTTP 422 при явном требовании параметра) — без выдачи всех состояний тенанта неограниченно.

#### Scenario: Состояния для группы инцидентов

- **WHEN** выполняется GET `/api/escalations/v1/{tenant}/incidents/state?incident_ids=A,B,C`, где у A и C есть эскалация, а у B — нет
- **THEN** ответ содержит элементы для A и C с их `current_tier` и `status`, а элемента для B нет

#### Scenario: Все запрошенные инциденты без эскалации

- **WHEN** запрошены `incident_ids`, ни у одного из которых нет состояния эскалации
- **THEN** возвращается HTTP 200 с пустым массивом

#### Scenario: Изоляция тенанта

- **WHEN** в `incident_ids` передан идентификатор инцидента другого тенанта
- **THEN** его состояние не включается в ответ
